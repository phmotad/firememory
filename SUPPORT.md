## Diagnose first

Before opening an issue, run the built-in diagnostics:

```sh
fquery doctor
fquery models list
fmem stats
```

## Bug reports and feature requests

Use [GitHub Issues](https://github.com/phmotad/firememory/issues).

Include:

- OS and architecture (`uname -sm` or `winver`)
- FireMemory version (`fmem version`)
- What you did, what you expected, what happened
- Output of `fquery doctor` (if relevant)

## Security issues

Do **not** open a public issue for security vulnerabilities. See [SECURITY.md](SECURITY.md).
