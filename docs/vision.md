# Vision

## Purpose

FireMemory is a local-first Brainfile for AI agents.

It stores operational memory in a single portable `.fbrain` file.

## Product Thesis

FireMemory is not SQL, not a generic vector database, and not just a RAG layer.

It offers cognitive operations:

- remember
- recall
- relate
- forget
- consolidate
- get_context
- explain
- sync

## Why It Matters

The MVP wins through:

- portability
- deduplication
- traceability
- context recovery
- local-first operation

## Non-Goals

The MVP does not optimize for:

- extreme performance
- distributed storage
- custom ANN indexes
- generative routing layers

FireQuery comes later, after FireMemory Core is functional.
